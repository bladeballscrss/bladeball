## 🛠️ Troubleshooting Common Issues

❌ **Script doesn't execute after pasting**
The most common cause is an outdated executor that can't handle newer Lua syntax. Update your executor to the latest build, restart Roblox, and try injecting the Blade Ball Script again.
**Solution:** Update your executor, clear its cache folder at `C:/Users/YourName/AppData/executor_cache/`, and re-inject.

❌ **GUI doesn't appear in-game**
This usually means the script loaded but the UI render call was blocked. Some executors need an extra step to allow drawing functions.
**Solution:** Enable `Drawing` or `UI Library` support in your executor settings, then re-run the Blade Ball Script.

❌ **Auto Parry misses fast balls**
If the ball speed exceeds your configured delay, parries will fire too late. This happens more often in later rounds.
**Solution:** Lower `ParryDelay` to `50–70`ms in the Blade Ball Script config and make sure `PredictionBuffer` is set to at least `3`.

❌ **Skin Changer shows default skin**
The Blade Ball Script Skin Changer relies on asset IDs that can shift when the game updates its cosmetic catalog.
**Solution:** Pull the latest version of the script — updated asset IDs are bundled with every new release.

❌ **High ping or rubber-banding after injection**
Heavy Auto Spam loops can flood the remote event queue on slow connections.
**Solution:** Reduce the spam interval in the Auto Spam Blade Ball Script settings or disable modules you aren't actively using.

❌ **Kicked from server with error code**
Server-side checks may flag unusual parry patterns during peak detection windows.
**Solution:** Enable `Natural Mode` in the Blade Ball Script GUI — it adds randomized delays to mimic human input timing.
